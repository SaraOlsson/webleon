Introduction

The thought of procedural generation has fascinated me since I first heard about it. I mean.. imagine walking through a virtual environment you set up yourself, and bump into creatures and scenery you did not explicitly created or placed there. To generate the meshes would be one step, and representing their surfaces another. This project focuses on the latter. In particular, methods for having the surface varying based on the surroundings are discussed. Further, as an UV-map would not be available for procedural generated meshes, one interesting aspect is to, despite that, achieve a varying surface based on area on the mesh. While a non-procedural mesh is used for testing, it can be applied for a procedural created mesh as well and I look forward to do so. To really point out the benefits of using a procedural shader, I chose to imitate the skin of a chameleon. This animal has the interesting capability of camouflaging according to its surrounding or mood, depending on species. Further, it has detailed cell pattern in its skin which opens up for the possibility to play around with different noise functions.

Aim

The purpose of this project is to achieve a procedural texture which colors depend on the surrounding. A control panel should be be available to the end user, for creative adjustments regarding parameters such as cell size, color variations and amount of procedural bump mapping.

To be used: Unity + HLSL shader code.

Theory

Characteristics of chameleons

First, because animals are cool, let us quickly cover some main characteristics of the chameleon skin which are to be imitated. In fact, the most outer skin is transparent and underneath are small “crystals” which colors are controlled by the chameleons hormones. However a virtual version works just fine with setting the final color directly, no layers needed. Like in real life, the transition is smooth and has a duration of about a second or two and this is a behaviour to keep.

UV-maps

A UV map can be interpreted as the 3D model stretched out into a flat 2D image, a procedure often referred as UV-unwrap. Each face in the polygon then has a corresponding face on the UV map and thus it's possible to attach attributes for different parts of the mesh easily. A common use of the UV map is to place 2D textures according to the flattened faces, while another is to mark areas with values representing other characteristics. As for example, these characteristics could correspond to the length, thickness or bend direction of fur. Also, a few integer values can be used to label areas as “leg”, “tshirt”, or “hair”, to then attach separate surface attributes in the shader based on the label. Especially this last example would be useful in the creation of a procedural texture. when the object has distinct variations in different areas.

Noise

Gradient noise is a technique used to produce natural structures in computer graphics, in particular to obey for the irregularity in the majority of all natural objects - things are simply not flat and exact. Grass grows in tufts, dust covers surfaces and kitchen clinker has details of imperfections. The list of examples can continue infinitely. A fragment shader processes each fragment individually and with the same code, yet the desired output should be continuous and coherent in its entirety. Gradient noise has proven its ability to meet these needs. The first known implementation of a gradient noise function is Perlin noise, which since then has increased the appearance of realism in computer graphics. Along with the classic Perlin noise, further developments has led to similar implementations as Simplex noise, Worley noise (Cellular Noise), and Voronoi noise. Common for all these implementations is the necessity that the positions used in the fragment shader are transformed to local coordinate space, as the chameleon is suppose to move.

Classic Perlin noise

The main idea of Perlin noise is to first generate regularly spaced points in space, each with function value zero and with a pseudo-random gradient assigned. Then, a continuous function is retrieved with an interpolation between the points given the gradients. Similarly for noise in higher dimensions, the points are regularly spaced in each direction for the dimension in question and interpolation is computed between the closest points. As the gradients are pseudo-random, the same result will be achieved every time the program runs with the same parameters which is often highly desired.

Cellular Noise

Cellular Noise is based on distance fields, the distance to the closest one of a set of feature points. Let's say we want to make a distance field of 4 points. What do we need to do? Well, for each pixel we want to calculate the distance to the closest point. That means that we need to iterate through all the points, compute their distances to the current pixel and store the value for the one that is closest.
